# Hackathon Notes

## Track Alignment

- Primary track: Real-World Asset (RWA) & Business Workflows
- Secondary track: Data, Analytics & Ecosystem Dashboards

## Why This Is Canton-Native

Compass addresses a privacy-native institutional workflow problem. Canton is relevant because selective visibility, role-aware workflows, and auditable shared processes are central to how benchmark contribution and institution-specific intelligence should work.

## What Is Mocked

- frontend data is demo-oriented
- API responses are static mock payloads
- TEE and attestation are represented architecturally, not implemented
- Canton package is a workflow skeleton, not a live deployment

## What Is Architectural

- private contribution model
- contribution quality and trust-weighted aggregation
- deterministic benchmark computation
- institution-scoped result delivery
- Canton-based audit and workflow framing
